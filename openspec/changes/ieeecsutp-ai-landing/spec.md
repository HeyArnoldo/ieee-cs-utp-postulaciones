# Spec: ieeecsutp-ai-landing

## Requirements

### R-01: Form collection
The SPA MUST collect all 7 fields from the original form:
- Full name (Nombres y Apellidos) — text
- Career (Carrera) — select from predefined list
- Cycle (Ciclo) — number 1–10
- Institutional email (Correo UTP) — text ending in @utp.edu.pe
- Personal email (Correo personal) — any valid email
- WhatsApp number — phone
- IEEE membership status (Es miembro IEEE) — yes/no
- IEEE Membership ID — conditional, only if member
- Area of interest (Comunidad/Comité) — select
- Sub-area (Línea/Departamento) — multi-select

### R-02: Dynamic AI follow-up
After the user provides their motivation answer, the system MUST:
1. Call `/api/repregunta` with the motivation text
2. Display a dynamic follow-up question generated by gpt-4o-mini
3. Collect the user's answer to that follow-up

### R-03: Persistence
On form submission, `/api/postular` MUST:
1. Write ALL form fields to the "Voluntarios IEEE CS UTP" Notion DB as a new page
2. Write the dynamic Q&A pair (question + answer) and motivation to the Notion page BODY
3. Return success/failure to the client

### R-04: AI must never block persistence
If the OpenAI call fails (timeout, rate limit, API error), the system MUST:
- Still persist the form data to Notion
- Log the error server-side
- Show the user a generic (non-AI) result message

### R-05: Personalized result
On success, the SPA MUST display a personalized result message generated by gpt-4o-mini (or a fallback message if AI fails) instead of the current hardcoded 87/100.

### R-06: Field mapping
The form option values DO NOT map 1:1 to Notion select values. The API MUST use a tested mapping layer to translate form values to valid Notion property values.

## Key Scenarios

### Scenario A — Happy path
1. User fills all 7 sections
2. Dynamic follow-up question appears after motivation
3. User answers follow-up
4. User submits
5. Data persists to Notion
6. Personalized result displayed

### Scenario B — AI failure during follow-up
1. User fills motivation
2. `/api/repregunta` fails
3. A fallback generic question is shown
4. User answers
5. Submission proceeds normally

### Scenario C — AI failure during result generation
1. User submits form
2. Notion write succeeds
3. AI result generation fails
4. Generic result message shown
5. Data is still in Notion

### Scenario D — Full AI failure
1. User submits
2. Notion write succeeds
3. Both AI calls fail
4. Generic messages shown throughout
5. Data is safe in Notion
